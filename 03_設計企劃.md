## 設計規範｜Visual Brief

### 色彩系統

| 角色 | Hex | 用法 |
|---|---|---|
| 主背景底色 | `#EBE9E5` | 全站底色 |
| 深色區塊 | `#0B1931` | Hero、Footer、見證引言區 |
| 品牌藍（強調） | `#013EC4` | Logo、數字、icon、連結 |
| 品牌金（CTA） | `#CC9901` | 主要按鈕、金橫條裝飾 |
| 深炭黑（文字） | `#1C1C1E` | 所有內文 |
| 點綴白 | `#FFFFFF` | 卡片、反白區 |

> 品牌藍 `#013EC4` 僅作點綴強調色，不大面積使用。深色滿版區塊一律用 `#0B1931`。
> CTA 按鈕：金色底 `#CC9901` + 深炭黑文字 `#1C1C1E`，對比度 5.7:1。

### 字型系統

| 用途 | 字型 |
|---|---|
| 中文標題 | Noto Serif TC Bold（明體） |
| 中文內文 | Noto Sans TC Regular（黑體） |
| 數字 / 英文 | DM Serif Display |

### 元件規範

| 元素 | 規格 |
|---|---|
| 容器 / 區塊 | 直角（border-radius: 0px） |
| 主要 CTA 按鈕 | Pill（border-radius: 24px），金色底 + 深炭黑文字 |
| 次要按鈕 | Pill（border-radius: 24px），藍色底 + 白色文字 |
| 裝飾線條 | 2px 實線，金色 `#CC9901` |
| 陰影 | 無（全站扁平） |

## 區塊 1｜Hero

**創意呈現概念**
三個信任錨點以「認證徽章（badge）」樣式呈現，橫排並列。每個 badge 搭配對應小 icon：
- 德國紅點設計獎得主 → 獎杯或獎章 icon
- 累積 30+ 位優質房客 → 人頭群組 icon
- 近半數外商或海外背景 → 地球或飛機 icon

**手機版排版規格**
- 主標題：佔滿螢幕寬度，最多兩行
- 副標題：三個短句各自換行
- 信任錨點 badge：橫排三個，螢幕過窄則換為兩行排列
- Sticky CTA 按鈕：固定在頁面底部，全程可見

---

## 區塊 2｜環境趨勢

**創意呈現概念**
三張卡片改為「稅務警示通知」風格——仿政府公文或稅單外觀，加上警告 icon，標題粗體強調，製造「官方通知」的視覺語氣。讓屋主潛意識感受「這是真實正在發生的事」，而非業者說詞。

**手機版排版規格**
- 三張卡片垂直堆疊，每張接近半屏高度
- 說明文字預設折疊（Accordion），點擊標題展開
- 引導句獨立一行，與卡片之間有明確留白

---

## 區塊 3｜包租代管科普

**創意呈現概念**
三欄對照改為「服務分級 tier 卡」風格，參考訂閱方案視覺語言。最右欄（包租代管）視覺強度最重（有邊框或標記），三條條件（招租、保證收租、租務管理）以 ✓ / ✗ 標示。最後一張包租代管三個全 ✓，形成自然視覺收尾。

**手機版排版規格**
- 三張卡片橫向滑動（Swipeable cards），每次顯示一張
- 順序：房仲 → 包租 → 包租代管（由少到多功能）
- 底部加滑動指示點（dot indicators）
- 最後一張（包租代管）建議為預設顯示位置

---

## 區塊 4｜現在 vs. 委託後的對比

**創意呈現概念**
五組對比改為「手機對話紀錄」風格，參考 Line 聊天介面：
- 左側灰色對話泡泡：屋主的煩惱（「以前」那一欄）
- 右側藍色對話泡泡：交給我們之後的狀態（「現在」那一欄）
目的：借用台灣日常最熟悉的溝通介面，讓內容感覺貼近生活。

**手機版排版規格**
- 五組對話泡泡依序垂直排列，使用者向下捲動逐組閱讀
- 每組之間留足間距，一次消化一組

---

## 區塊 5｜整理前後對比

**創意呈現概念**
改為「左右拖曳顯示」的 before/after reveal slider：使用者用手指左右拖動，即時看見照片從 Before 漸變為 After。讓屋主「親手」看見改變，參與感遠高於靜態前後對比。技術上為標準 before/after image slider，前端實作常見。

**手機版排版規格**
- 每組 reveal slider 佔全螢幕寬度
- 多組案例以橫向 Carousel 呈現，滑動切換
- 底部加滑動指示點
- 照片比例每組統一，以 4:3 橫式為主
- 影像規格（尺寸、格式、解析度）待確認素材後另行定義

## 區塊 6｜屋主見證

**創意呈現概念（兩個方向擇一）**

方向 A：Google 地圖評論風格
仿 Google Maps 評論卡片：屋主頭像、五顆星、地區標記、「已驗證屋主」標籤。借用 Google 評論的視覺公信力。

方向 B：Line 訊息截圖風格
仿真實 Line 對話截圖：頭像、名稱、訊息泡泡、時間戳記、已讀記號。對台灣用戶極度熟悉，第一眼感受「這是真實傳來的訊息」。

**手機版排版規格**
- 三則見證以橫向滑動 Carousel 呈現，每次顯示一則
- 卡片高度由文字決定，不固定高度
- 底部加滑動指示點
- 日後新增見證，文字長度建議 50–80 字

## 區塊 7｜服務流程

**創意呈現概念**
在六步驟之前，先用大字強調：「你只需要做一件事——打開 Line，其餘全部由我們來。」
接著展開六步驟，每個步驟旁標注「屋主」或「我們」負責，讓屋主一眼看出六步中他只需負責第一步。

**手機版排版規格**
- 前置大字字體明顯大於正文
- 六步驟垂直時間軸，左側步驟編號圓圈，右側步驟名稱（粗體）加說明
- 步驟 2「免費到府評估」加視覺強調
- 六步驟一次全部顯示，不折疊

---

## 區塊 8｜試算表格

**創意呈現概念**
四個輸出結果差異化處理：
- 節稅效益與補稅罰鍰：放大顯示，搭配強調色，作為視覺重心
- 自行出租與包租代管所得稅：正常大小，作為計算依據

微互動：使用者滑到試算區塊時，輸入欄位以預設值動態跳出；結果卡片隨後依序淡入，讓屋主感受「這個東西是活的」。

**手機版排版規格**
- 輸入區三個欄位垂直排列
- 月租金欄位點擊自動觸發數字鍵盤
- 稅率使用系統原生下拉選單（`<select>`）
- 使用者改數字時結果即時更新，不需按計算按鈕
- 四個結果各自獨立一張卡片，垂直排列
- 試算工具 B 預設折疊

## 區塊 9｜FAQ

**創意呈現概念**
標準 Accordion，每題前加「Q」標記。最後一題「現在不急」展開答案後，答案最後一行加輕量行動引導（如「先預約看看也無妨 →」），自然銜接 Final CTA。

**手機版排版規格**
- 全部預設收合
- 每個問題列點擊區域高度至少 48px
- 問題文字不截斷，允許換行
- 展開動畫 200–300ms
- 「現在不急」排在最後一題

---

## 區塊 10｜Final CTA

**創意呈現概念**
三個保證改為三個「保證印章（badge）」樣式：圓形或方形 badge，各自搭配小 icon（零元符號、無鎖鏈、日曆打叉），三個橫排並列。視覺效果類似「三重保障標章」。

**手機版排版規格**
- 三個 badge 橫排，螢幕過窄（如 iPhone SE）則換為三行垂直排列
- CTA 按鈕佔螢幕寬度至少 80%、高度 52px 以上
- 此區塊視覺同時作為頁面底部 sticky bar 的樣式來源

**手機版排版規格**
- 文字居中排列，字體比正文小一階
- 補充公司名稱、統編、包租代管業執照字號（視取得進度補入）

## 整體手機版技術規格摘要

| 元素 | 規格 |
|---|---|
| Sticky CTA | 固定在頁面底部，z-index 最高層，全程可見 |
| Accordion | 預設收合，展開動畫 200–300ms |
| Carousel | 支援滑動手勢，底部附 dot indicators |
| 輸入框 | 數字型欄位自動觸發數字鍵盤 |
| 下拉選單 | 使用系統原生 `<select>`，不用自製選單 |
| 點擊區域 | 所有可點擊元素最小高度 48px |
| 圖片比例 | Before/After 照片統一 4:3 橫式 |
| 試算更新 | 輸入值改變時即時重新計算，不需按鈕觸發 |
